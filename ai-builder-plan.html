Sprint 2 — What I Built
Deliverable 01 · Interface
Live kanban board with 6 automation ideas as draggable cards — status updates persist to Notion in real time.
Live ↗
What was built
Deployed ai-builder-dashboard.html — a fully branded 47G kanban board with four columns (Suggested, To Build, In Progress, Done). Each card represents one automation from the roadmap with priority badge, description, and drag-to-move behavior.
How it works
On load, the board fetches live data from the Notion backend via /api/roadmap. Dragging a card to a new column fires a PATCH request that updates the Status field on the Notion page directly — no sync required, no export step.
Quick Win · 1 session
Open Dashboard ↗
Deliverable 02 · Backend
Notion database wired as the live source of truth for all automation work — no more status living only in a static HTML file.
Live ↗
What was built
Created the AI Automation Roadmap database in Notion with fields for Name, Description, Priority, Status, Submitted By, and Submitted At. Built a serverless API (/api/roadmap.js) deployed to Vercel with GET (fetch all rows) and PATCH (update Status by page ID) endpoints.
How it works
The API reads NOTION_TOKEN and NOTION_DATABASE_ID from Vercel environment variables — credentials never touch the frontend. Notion Status values map cleanly to board columns (e.g. "In Progress" → in-progress), so the board always reflects the real state of the roadmap.
Quick Win · 1 session
Live API ↗
Deliverable 03 · Agent
Automation advisor agent reads the live roadmap and session history, then delivers a 60-second briefing — priorities, stalls, and one automation to add next.
Runs on demand
What was built
A Claude Code agent that pulls current card statuses from the Notion API, reviews Claude Code session history for stalls and patterns, and outputs a structured briefing: what's moving, what's stuck, and one concrete automation recommendation to add to the roadmap.
How it works
Trigger it any time before a build session or weekly review. The agent returns a plain-language briefing in under 60 seconds — no dashboard to open, no spreadsheet to update. The recommendation feeds directly back into the Notion roadmap if you accept it.
